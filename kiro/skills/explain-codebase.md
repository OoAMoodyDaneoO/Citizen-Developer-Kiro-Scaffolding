# Explain This Codebase

Generate a plain-language explanation of this project for someone who is not a developer.

## What To Cover
- What does this application do? (one paragraph, no jargon)
- What are the main screens or pages a user would see?
- How does data flow through the application? (explain like a flowchart in words)
- What external services does it connect to? (databases, APIs, auth providers)
- What happens when a user performs key actions? (login, submit a form, view data)

## How To Write It
- Use everyday language — avoid technical terms or explain them in parentheses
- Use analogies where helpful
- Keep sentences short and clear
- Use bullet points for lists of features or steps
- Include a simple diagram description if the architecture is complex

## Output
Create a CODEBASE-GUIDE.md file in the project root that any non-technical stakeholder could read and understand what this project does, how it works at a high level, and what state it is in.
