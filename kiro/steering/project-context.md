# Project Context & Organizational Standards

This workspace is configured with organizational guardrails for safe prototyping. Whether you're a product manager, designer, analyst, or anyone building a prototype — Kiro will guide you through best practices automatically.

## Who This Is For

You don't need to be a developer. Describe what you want to build in plain language and Kiro will:

- Generate code that follows your organization's security standards
- Use approved UI frameworks and component libraries
- Handle authentication and authorization correctly
- Structure the project for easy handoff to engineering teams
- Run automated checks on everything it produces

## How To Get Started

1. Use **Specs** to describe your feature — Kiro will create requirements, a design, and implementation tasks
2. Use **Chat** to ask Kiro to build things conversationally
3. Hooks will automatically check your code as files are created and edited
4. If you're unsure, just ask — Kiro knows the rules and will follow them

## Key Principles

- Security is non-negotiable — secrets, auth, and data handling are enforced automatically
- Approved libraries and frameworks only — no random npm packages
- Accessible by default — UI components meet accessibility standards
- Production-ready patterns — even for prototypes, the foundations should be solid
